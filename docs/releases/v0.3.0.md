# Triluga 0.3.0

Data: `2026-04-08`

## Resumo

A versão 0.3.0 prepara o Triluga para um deploy mais direto no Render, melhora a configuração dinâmica entre frontend e backend e reduz atritos no fluxo de cadastro.

## Entregas principais

- blueprint `render.yaml` com frontend estático, backend Node e banco Postgres no Render
- backend passando a resolver `APP_URL` automaticamente a partir do ambiente do Render
- frontend deixando `assets/app-config.js` fora do cache do service worker
- geração de runtime config inferindo a origem pública do backend sem precisar de ajuste manual
- cadastro com telefone vazio por padrão e correção na formatação ao editar números com `+55`
- novos testes para `AppComponent` e `RegisterPageComponent`

## Impacto esperado

- deploy inicial mais rápido e com menos configuração manual
- menor chance de frontend apontar para URL errada em produção
- menos atrito no preenchimento do cadastro em dispositivos móveis
- maior confiança para evoluir bootstrap e auth flow com cobertura automatizada
